# v1.4.0 Aggressive Kaggle Workflow / 激进 Kaggle 提分流程

This release adds the planned aggressive v1.4 experiment infrastructure. It is designed for Kaggle GPU sessions where training may need to continue across multiple submissions.

本版本新增 v1.4 激进提分实验基础设施，面向可能需要多次提交续跑的 Kaggle GPU 训练场景。

The full workflow was executed on Kaggle from commit `e893e5a2a73c4c3c248abacdc39171375f38f59a`. All four planned models finished training. The run does not replace the default inference model.

完整流程已基于 commit `e893e5a2a73c4c3c248abacdc39171375f38f59a` 在 Kaggle 上运行，四个计划模型均完成训练。本次运行不替换默认推理模型。

## Main Additions / 主要新增

- Higher-capacity experiment runner: U-Net++ EfficientNet-B4, 512 fine-tune, DeepLabV3+ EfficientNet-B4, and optional EfficientNet-B5.
- Time-safe training: `training.max_runtime_minutes` and `training.safe_stop_minutes` save `last_model.pth` before the session runs out.
- Fine-tune resume controls: `resume_optimizer` and `resume_training_state` allow loading weights without inheriting old optimizer/scheduler history.
- OOM fallback: the v1.4 runner reduces batch size and increases gradient accumulation when CUDA OOM is detected.
- TTA and post-processing evaluation: flip/scale TTA, small-component removal, and hole filling.
- Optional hard-example replay infrastructure through `data.sample_weights_csv`.
- Macro-per-image primary metrics for TTA/ensemble evaluation, with pixel-level results retained under explicit `micro_*` fields.
- Batch-size override and failure logs for memory-heavy TTA/ensemble evaluation; partial post-processing failures no longer discard completed training metrics.

## Completed Results / 已完成结果

The table below uses the same per-image macro aggregation as `evaluate.py` and the v1.3 reference:

下表使用与 `evaluate.py` 和 v1.3 reference 相同的逐图 macro 聚合方式：

| Candidate | Test Dice | External Dice | Test Boundary F1 | External Boundary F1 | Training time |
| --- | ---: | ---: | ---: | ---: | ---: |
| v1.3 `contrast_aug_bce_dice` reference | 0.864766 | 0.924386 | 0.437615 | 0.593236 | - |
| U-Net++ EfficientNet-B4 448 | 0.860389 | **0.925023** | 0.386077 | 0.506854 | 50m 30s |
| U-Net++ EfficientNet-B4 512 fine-tune | 0.862619 | 0.918162 | 0.359950 | 0.418853 | 41m 45s |
| DeepLabV3+ EfficientNet-B4 448 | 0.859388 | 0.913495 | 0.392166 | 0.432164 | 2h 50m 49s |
| U-Net++ EfficientNet-B5 448 | **0.864376** | 0.919449 | **0.395422** | **0.506858** | 1h 19m 46s |

The 448 EfficientNet-B4 model produced the best external Dice, while EfficientNet-B5 produced the best v1.4 single-model internal-test Dice. Neither model improved both primary splits, and all v1.4 candidates had lower Boundary F1 than the v1.3 reference.

448 EfficientNet-B4 获得最佳 external Dice，EfficientNet-B5 获得 v1.4 单模型最佳 internal-test Dice；但没有模型同时改善两个主要划分，且全部 v1.4 candidate 的 Boundary F1 均低于 v1.3 reference。

The original run's TTA evaluator aggregated Dice/IoU over all pixels. Under that micro aggregation, the 512 fine-tune improved test Dice from `0.866804` in fast mode to `0.876619` with multi-scale flip TTA and post-processing. This is a within-run gain of `+0.009815`, but it is not a valid macro comparison against v1.3. The evaluator has been corrected for future runs.

原始运行的 TTA evaluator 按全部像素聚合 Dice/IoU。在该 micro 口径下，512 fine-tune 的 test Dice 从 fast 模式的 `0.866804` 提升至多尺度 flip TTA + 后处理的 `0.876619`，同口径提升 `+0.009815`；但该值不能作为与 v1.3 macro 指标的有效对比。后续运行的 evaluator 已修正。

## Kaggle Commands / Kaggle 命令

```bash
python notebooks/kaggle_aggressive_v1_4.py --debug
python notebooks/kaggle_aggressive_v1_4.py
```

Retry previously failed stages:

```bash
python notebooks/kaggle_aggressive_v1_4.py --retry-failed
```

This mode also refreshes stale TTA/ensemble outputs after an evaluator schema change while reusing completed training checkpoints.

该模式也会在 evaluator schema 更新后刷新旧 TTA/ensemble 输出，并复用已完成训练的 checkpoint。

## Replacement Rule / 默认模型替换规则

Replace the default model only if internal test Dice improves by at least `+0.012`, external Dice does not drop by more than `0.005`, and Boundary-F1 does not decline. TTA or ensemble results should be published separately as `best_accuracy` if they materially improve accuracy but increase inference time.

只有当 internal test Dice 至少提升 `+0.012`、external Dice 下降不超过 `0.005` 且 Boundary-F1 不下降时，才替换默认模型。若 TTA 或 ensemble 明显提分但增加推理耗时，应单独作为 `best_accuracy` 配置发布。

## Replacement Decision / 替换结论

- Replace default model: `False`.
- Best v1.4 single-model test Dice: `0.864376`, a change of `-0.000390` versus the v1.3 reference.
- Best v1.4 external Dice: `0.925023`, a change of `+0.000637` versus the v1.3 reference, but its test Dice and Boundary F1 declined.
- The published TTA values use a different aggregation and therefore cannot satisfy the replacement gate until re-evaluated with corrected macro metrics.

- 是否替换默认模型：`False`。
- v1.4 单模型最佳 test Dice 为 `0.864376`，相对 v1.3 reference 变化 `-0.000390`。
- v1.4 最佳 external Dice 为 `0.925023`，相对 v1.3 reference 提升 `+0.000637`，但该模型的 test Dice 和 Boundary F1 均下降。
- 已发布 TTA 数值使用不同聚合口径，在使用修正后的 macro 指标重新评估前，不能用于通过替换门槛。
