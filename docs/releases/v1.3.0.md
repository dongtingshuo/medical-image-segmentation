# v1.3.0 Low-Contrast Specialist Workflow / 低对比度专项流程

This release adds and reports a Kaggle workflow for testing whether low-contrast-specific augmentation and loss choices improve segmentation on low-contrast images. The completed results do not satisfy the replacement criteria, so the default inference checkpoint remains unchanged.

本版本新增并报告 Kaggle 低对比度专项流程，用于验证低对比度增强和 loss 选择是否能改善低对比度图像分割效果。完整结果未满足替换标准，因此默认推理 checkpoint 保持不变。

## Variants / 实验变体

- `control_bce_dice`: current high-accuracy recipe with BCE + Dice.
- `contrast_aug_bce_dice`: low-contrast augmentation with BCE + Dice.
- `contrast_aug_focal_dice`: low-contrast augmentation with Focal + Dice.
- `contrast_aug_tversky`: low-contrast augmentation with Tversky loss.

## Main Commands / 主要命令

Debug run:

```bash
python notebooks/kaggle_low_contrast_v1_3.py --debug
```

Full run:

```bash
python notebooks/kaggle_low_contrast_v1_3.py
```

If Kaggle stops at the GPU time limit, submit the same command again. Completed variants are skipped through `completed.json`, and unfinished variants resume from `checkpoints/last_model.pth` when available.

如果 Kaggle 因 GPU 时间限制停止，重新提交同一命令即可。已完成变体会通过 `completed.json` 跳过，未完成变体会在可用时从 `checkpoints/last_model.pth` 继续训练。

## Replacement Criteria / 替换默认模型标准

The default model should be replaced only if the best v1.3 variant satisfies all conditions below:

只有当最佳 v1.3 变体同时满足以下条件时，才建议替换默认模型：

- Internal test low-contrast Dice improves by at least `+0.02`, or low-contrast Recall improves by at least `+0.03`.
- Overall internal test Dice drops by no more than `0.01`.
- Visual samples do not show all-black masks, all-white masks, or large false-positive regions.

## Completed Results / 已完成结果

The full Kaggle run completed at commit `f63353a9f6f5ed81a52e923ad1a2b1894a78d0c4` with image size `384`, batch size `8`, and four variants. The generated comparison report selected `contrast_aug_bce_dice` as the best variant but did not recommend replacing the default model.

完整 Kaggle 运行已在 commit `f63353a9f6f5ed81a52e923ad1a2b1894a78d0c4` 完成，输入尺寸为 `384`，batch size 为 `8`，共比较四个变体。生成的对比报告选择 `contrast_aug_bce_dice` 为最佳变体，但不建议替换默认模型。

Decision:

- Default replacement: `False`.
- Reason: internal-test low-contrast Dice improved by only `+0.000800`, and internal-test low-contrast Recall improved by `+0.009289`; both are below the configured replacement thresholds.
- Visual samples did not show all-black or all-white masks in the downloaded sample set.

- 是否替换默认模型：`False`。
