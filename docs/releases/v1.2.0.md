# v1.2.0 Research Workflow Upgrade / 研究流程升级

This release adds and reports a small-budget Kaggle research workflow for robustness analysis. It does not publish new model weights or replace the current default inference checkpoint.

本版本新增并报告小预算 Kaggle 研究增强流程，用于稳健性分析。它不发布新的模型权重，也不替换当前默认推理 checkpoint。

## Highlights / 主要变更

- 3-fold cross-validation helpers with strict image/mask stem matching and leakage checks.
- Encoder comparison runner for high-capacity models such as U-Net++ with EfficientNet-B3 and ResNet34 encoders.
- Subgroup analysis by lesion size and image contrast, using per-sample segmentation metrics.
- Statistical summary utilities with mean, sample standard deviation, and 95% confidence intervals.
- Kaggle research script `notebooks/kaggle_research_v1_2.py` for end-to-end execution.
- Sanitized research artifact packaging support for v1.2 outputs, excluding materialized fold data and medical image files.
- Unit tests for cross-validation, subgroup analysis, and statistical analysis.

The best encoder by validation Dice was EfficientNet-B3. The v1.2 threshold search selected `0.55` for the encoder-comparison checkpoint, with Dice `0.875356` and IoU `0.778340`. This result does not materially exceed the existing released default model workflow, so `configs/final_model.yaml` remains the default local inference config.

验证集 Dice 最优 encoder 为 EfficientNet-B3。v1.2 阈值搜索为 encoder-comparison checkpoint 选择 `0.55`，对应 Dice `0.875356`、IoU `0.778340`。该结果没有显著超过现有已发布默认模型流程，因此 `configs/final_model.yaml` 仍作为默认本地推理配置。

Subgroup analysis indicates that low-contrast images are consistently harder than high-contrast images. On the internal ISIC 2017 test split, low-contrast Dice was `0.832174`, while high-contrast Dice was `0.902537`. On the external ISIC 2018 split, low-contrast Dice was `0.888860`, while high-contrast Dice was `0.950750`.

子组分析表明低对比度图像持续比高对比度图像更难。在 ISIC 2017 内部测试集上，低对比度 Dice 为 `0.832174`，高对比度 Dice 为 `0.902537`。在 ISIC 2018 外部集上，低对比度 Dice 为 `0.888860`，高对比度 Dice 为 `0.950750`。
